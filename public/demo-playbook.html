01 — WHAT IS BALNCE

Balnce is an AI-powered accounting platform built specifically for Irish businesses. It automates bookkeeping, VAT returns, and tax return preparation (CT1, Form 11, RCT) by processing data that general AI models have never seen — private bank statements, paper receipts, and Irish-specific tax rules.

General-purpose tools (Xero, QuickBooks) require manual categorisation and don't understand Irish tax law. Irish tools (Big Red Cloud, Surf Accounts) handle compliance but have zero AI. Balnce combines a codified Irish tax rules engine with AI-powered categorisation, OCR receipt scanning, and a live AI chat assistant powered by Claude Sonnet.

Core Insight: The difference isn't a better model — it's better data. Rules engines + structured data beat RAG for compliance. Cross-referencing private financial data creates value that no general model can replicate.

03 — THE OPENCLAW INTEGRATION

Balnce computes the tax. But what if an AI agent could also file it?

We built a prototype integration with OpenClaw — an open-source AI agent created by Peter Steinberger (recently acquired by OpenAI). OpenClaw can control a browser, read data from APIs, and perform multi-step tasks autonomously.

The Pipeline

Ingest → Categorise → Cross-Reference → Compute → File

1. Upload bank CSV → Balnce web app
2. Auto-categorise all transactions → Tax rules engine
3. Compute CT1 return → Real-time hooks
4. Open Revenue CT1 form → OpenClaw browser agent
5. Fill every field automatically → OpenClaw + Balnce API
6. Stop. Human reviews. Human submits.

Key Takeaway: The AI agent doesn't replace the accountant — it does the data entry. The human reviews and approves. AI prepares, human files.

05 — SECURITY & COMPLIANCE

We built this to show what's possible. We're not shipping it blindly. There are four real concerns:

ConcernRiskOur Position
ROS Terms of Service Revenue hasn't approved bot-driven submissions. Automating ROS may violate ToS. Demo shows capability only. Production requires Revenue approval or official API.
Credential Handling ROS uses .p12 certificate auth. Agent access = full access to someone's tax life. We'd never store or transmit certs. Sandboxed session on user's own device only.
Liability Wrong number on a tax return — who's responsible? AI prepares, human files. Always stops before submission. Liability chain stays clear.
GDPR Tax data is sensitive. AI agent = additional data processor. Supabase is EU-hosted. OpenClaw runs locally. No tax data leaves the device.

Bottom line: The tax computation engine is live, accurate, and applying real Irish tax law. The form-filing is a prototype showing the direction of travel. We ship when the regulatory wrapper is in place.

07 — Q&A PREP

"Who is this for?"

Irish limited companies with 1–2 directors. Primarily trades (construction, electrical, plumbing) and small tech/SaaS companies. The ones paying €3k/year for an accountant to do data entry.

"How accurate is the auto-categorisation?"

Rule-based matching against Irish VAT and CT legislation — not a generic LLM guess. 200+ merchant pattern rules, receipt OCR, and Section 60 keyword analysis. Users can review and override. Anything uncategorised is conservatively flagged as a CT add-back until confirmed.

"Are you replacing accountants?"

We're replacing the 80% that's manual grunt work — data entry, bank reconciliation, basic compliance filing. Complex tax planning, audits, and advisory still need a human. We make the human's job easier, not redundant.

"What's the business model?"

SaaS subscription. The accountant is a €3,000/year cost that most small companies accept as unavoidable. We make it avoidable — or at least dramatically cheaper.

"How is this different from Xero or QuickBooks?"

They're general-purpose tools that require manual categorisation and don't understand Irish tax rules. Balnce is purpose-built for Irish CT1/VAT/RCT compliance. It applies the rules automatically — Section 60 VAT, CT add-backs, two-thirds rule, capital allowances. No accountant in the loop for the basics.

"What about the OpenClaw integration — is it real?"

The tax computation is real and live. The form-filling demo is a prototype showing the direction. We built it to demonstrate the end-to-end vision. Production deployment requires Revenue approval and a proper credential security framework — which we've outlined and are working toward.
